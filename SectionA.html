Section A: Fundamental Architectural Components and Methodologies

CONCEPT CARD 1

Aarya Yugant Mhatre

1. Text Preprocessing & Feature Engineering

Definition
The process of dividing a continuous block of text into distinct, structural individual sentences.
Purpose
To establish boundaries in unstructured text bodies, ensuring subsequent workflows handle semantic sentences rather than unstructured streams.
Working Principle
Utilizes rule-based regular expressions (detecting terminal punctuation) paired with unsupervised ML packages (like the Punkt tokenizer) trained to distinguish full-stops from internal abbreviations.
Examples
"Hello world! Welcome to NLP." → ["Hello world!", "Welcome to NLP."]
Advantages
Crucial structural step for accurate document parsing; prevents boundaries from leaking across separate distinct contexts.
Limitations
Fails on irregular shorthand, unpunctuated social media logs, and ambiguous domain acronym periods.
Applications
Document summarizing frameworks, structural machine translation pipelines, conversational routing frameworks.
Definition
Splitting a sequence of strings into smaller syntactic units called tokens (words, subwords, or characters).
Purpose
To isolate the atomic processing building blocks of text for statistical, matrix, or continuous vector operations.
Working Principle
Whitespace tokenization splits strings by spaces. Modern tokenizers (BPE, WordPiece) break words down recursively into subword elements to mitigate out-of-vocabulary anomalies.
Examples
"Natural Language Processing" → ["Natural", "Language", "Processing"]
Advantages
Standardizes textual inputs; subword variants cleanly handle structural lexical gaps.
Limitations
Aggressive tokenization scripts can strip key context or isolate idiomatic expressions incorrectly.
Applications
Lexical compilation passes, large language model vocabulary building blocks.
Definition
Converting all structural characters uniformly into a single casing layout, typically lowercased text.
Purpose
To reduce vocabulary redundancy by matching identical words written with different cases.
Working Principle
Maps character ASCII/Unicode entries directly to lower-case index values during processing.
Examples
["Learning", "LEARNING"] → ["learning", "learning"]
Advantages
Shrinks total tracking dimension footprint; guarantees start-of-sentence words map correctly.
Limitations
Distorts proper nouns and acronyms where case changes the absolute definition (e.g., "US" pronoun vs "US" country).
Applications
Information retrieval indexing setups, lookup engines.
Definition
Filtering out highly frequent structural words that carry minimal unique semantic weight.
Purpose
To filter out background syntactic noise, isolating high-information tokens.
Working Principle
Compares incoming tokens against a predefined dictionary table list (e.g., "the", "is", "at") and discards matching terms.
Examples
"This is sample text" → ["sample", "text"]
Advantages
Improves compute pipeline throughput; cuts down feature matrix constraints.
Limitations
Can disrupt sequence patterns, completely invert sentiment values, or erase relational clues (e.g., "Not good").
Applications
Document categorization matrices, key phrase extraction algorithms.
Definition
A crude, fast heuristic sequence that truncates word endings to resolve them to a common baseline root.
Purpose
To group morphological variants of a word under a single tracking token using algorithmic rules without lookup databases.
Working Principle
Applies structural rule-based processes (such as Porter's or Lancaster's rules) to slice suffix parts (like "-ing", "-ed") sequentially.
Examples
"running", "runs" → "run" (while "ran" might parse to "ran").
Advantages
Highly efficient; zero dictionary overhead; low execution footprint.
Limitations
Causes over-stemming mutations (e.g., "universe" and "university" to "univers") or under-stemming discrepancies.
Applications
Broad index search applications, live log query parsing setups.
Definition
A full structural morphological routine that converts a token to its linguistically valid canonical base form (lemma).
Purpose
To ensure correct base conversions while preserving structural, grammatical validity.
Working Principle
Cross-references terms against electronic lexical databases (like WordNet) combined with explicit Part-of-Speech context hooks.
Examples
"running", "ran", "runs" → "run"
Advantages
Linguistically exact; keeps entries readable and structurally valid.
Limitations
Slower; requires memory-intensive database hooks and contextual analysis passes.
Applications
Semantic analysis pipelines, advanced question-answering engines.
Definition
The systematic identification and extraction of completely non-textual structural items from raw document bodies.
Purpose
Cleans out layout formats, web markup code, and metadata that provide no linguistic context.
Working Principle
Applies regular expression patterns and parsing trees to locate and erase markup language sequences.
Examples
"<p>Content</p>" → "Content"
Advantages
Keeps structural layout syntax from polluting data statistics patterns.
Limitations
Poorly structured cleaning patterns risk discarding valid data payloads.
Applications
Web scraper pipelines, social media parsing systems.
Definition
Fixing typos, removing emoji artifacts, normalizing slang variations, and correcting character encodings.
Purpose
Converts unpredictable, messy text inputs into standardized data layouts.
Working Principle
Uses lookup translation tables, edit-distance spelling matches, and character set mapping rules (UTF-8).
Examples
"L0ve NLP!! 😊" → "Love NLP"
Advantages
Boosts text quality; minimizes out-of-vocabulary bugs caused by spelling inconsistencies.
Limitations
Risk of over-cleaning, where localized emphasis or nuance is accidentally stripped.
Applications
Helpdesk ticket parsing, social feedback analysis platforms.

2. Feature Engineering

Definition
A vector space representation model that tracks raw word frequencies within a document, completely discarding order and syntax structure.
Purpose
Converts unstructured raw text inputs into fixed-length numeric vectors for machine learning model pipelines.
Working Principle
Builds a master vocabulary matrix of all unique tokens across a dataset. Each document is mapped to a vector of length V, where each index records the local count of that specific term.
Examples
"cat sat cat" → {"cat": 2, "sat": 1}
Advantages
Simple to construct; clear to debug; excellent baseline classification feature set.
Limitations
Suffers from extreme vector sparsity; ignores word positioning ("not bad" and "bad not" output identical representations).
Applications
Spam classification filters, entry-level topic filters.
Definition
A contiguous, sequential tracking sequence of n items or words pulled from a given text profile.
Purpose
Captures local structural word sequences, phrase combinations, and relational co-occurrences within an index layout.
Working Principle
Slides an operational tracking window of n items across the text stream token-by-token, building linked sequence tuples.
Examples
"Deep Learning" (n=2) → Bi-gram: ("Deep", "Learning")
Advantages
Preserves critical context pairs (e.g., tracking phrases like "not guilty").
Limitations
Vocabulary tracking size scales exponentially as n grows ($V^n$), leading to extreme storage bottlenecks and matrix sparsity.
Applications
Next-word predictive systems, language structural identification systems.
Definition
A normalized statistical metric tracking how frequently a specific target token occurs within an individual text file.
Purpose
Quantifies the relative presence or density of a target term inside a localized document file context.
Working Principle
Calculated as the total occurrences of term t divided by the total sum of all tokens inside that target document matrix layout.
Examples
Word "AI" appears 5 times in a 100-word essay → TF = 0.05
Advantages
Linear calculation; easily scales based on localized density cues.
Limitations
Biased toward common structural words like "is", "the", which naturally dominate frequency tables.
Applications
Isolated document keyword relevancy tracking passes.
Definition
A measurement checking how rare or globally unique a target word is across the total corpus database.
Purpose
Penalizes ubiquitous terms that appear across all files while scaling up the weights of unique keywords.
Working Principle
Computed as the log of the total document count N divided by the count of files containing that target term.
Examples
Total files = 1000. "the" is in all 1000 → IDF = log(1) = 0. "Quantum" is in 10 → IDF = log(100) = 2.
Advantages
Filters global noise automatically; extracts highly informative terms.
Limitations
Requires recalculation across the entire global corpus whenever new document layers are added.
Applications
Search relevance systems, information extraction pipelines.
Definition
The mathematical product of local Term Frequency and Inverse Document Frequency, yielding a relative statistical weight.
Purpose
Represents documents as vectors that balance local occurrence density with global distinctiveness.
Working Principle
Combines both metrics multiplication-style: TF-IDF = TF × IDF.
Examples
TF = 0.05, IDF = 2 → TF-IDF Score = 0.10.
Advantages
Balances local density with global rarity; effectively reduces the influence of meaningless stop-words dynamically.
Limitations
Vectors remain sparse and high-dimensional; fails to capture semantic synonyms (e.g., treats "large" and "huge" as separate, unrelated weights).
Applications
Information retrieval scoring, baseline document recommendation systems.

CONCEPT CARD 2

Shraddha Sandip Salvi

1. Language Representation

Definition
A categorical vector transformation where every unique word in a vocabulary is represented as a distinct binary vector containing a single '1' and all other entries as '0'.
Purpose
To assign a unique, discrete mathematical coordinate identity to every word inside a known lexicon list.
Working Principle
Given a vocabulary length V, a word is assigned an index i. The representation is a vector of size V×1 with a '1' at index i and '0' everywhere else.
Examples
Vocabulary: ["apple", "banana", "cat"]. Vector for "banana" → [0, 1, 0].
Advantages
Extremely straightforward to execute; guarantees zero overlapping representations between words.
Limitations
Highly inefficient due to vector sparsity; orthogonal geometry means the dot product of any two words is '0', implying "king" and "queen" share exactly zero mathematical similarity.
Applications
Categorical initialization passes, baseline network target label parsing layouts.
Definition
A collective term for language modeling methodologies where words are mapped to low-dimensional, continuous dense vector spaces.
Purpose
To capture complex continuous semantic and syntactic relationships in a compressed matrix architecture.
Working Principle
Maps words to a dense continuous vector space of fixed size (e.g., 300 dimensions). Positions are optimized based on contextual co-occurrence patterns, ensuring similar words cluster closely together.
Examples
"king" - "man" + "woman" ≈ "queen" expressed within a dense float vector space.
Advantages
Highly compact (dense matrices); natively maps geometric distance to word meaning similarity.
Limitations
Traditional static variations assign a single vector to a word, ignoring multiple meanings (polysemy).
Applications
Neural network input representations, vector-similarity search engines.
Definition
A groundbreaking framework utilizing shallow two-layer neural networks to learn static dense word embeddings.
Purpose
To generate high-quality, dense semantic word vectors efficiently from vast unannotated text corpora.
Working Principle
Offers two architectures: Continuous Bag-of-Words (CBOW), which predicts a target word given context words, and Skip-Gram, which predicts context words given a target word. Uses negative sampling to handle loss calculation.
Examples
Outputs a 300-dimension float array for the word "coffee", clustering it near "tea" and "latte".
Advantages
Efficient training on massive datasets; captures excellent linear semantic analogies.
Limitations
Static representation; cannot dynamically adjust a word's vector representation based on changing surrounding context (e.g., "bank" of a river vs financial "bank").
Applications
Downstream NLP feature generation, recommendation systems.
Definition
An extension of the Word2Vec framework developed by Facebook that treats each word as a composite collection of subword character n-grams.
Purpose
To build resilient embeddings that natively resolve out-of-vocabulary (OOV) terms and capture morphological variations.
Working Principle
Breaks words into subword pieces. The final word vector is computed as the mathematical sum of its constituent subword character n-gram vectors.
Examples
An unseen word "sub-optimal" can still get an accurate vector by summing up sub-components like "sub" and "optimal".
Advantages
Cleanly handles out-of-vocabulary words; highly robust to typos, prefixes, and suffixes.
Limitations
Higher RAM and storage footprint due to tracking massive volumes of internal subword components.
Applications
Text classification in morphologically rich languages (e.g., German, Finnish), search queries with frequent typos.
Definition
Dynamic vector representations where the mathematical vector assigned to a word changes depending on its specific sentence context.
Purpose
To cleanly solve the issue of polysemy, ensuring identical words have completely distinct vector mappings based on context.
Working Principle
Employs deep multi-layer bidirectional architectures (like Transformers) to read the entire sequence of text at once, dynamically re-weighting a word's vector representation using attention mechanisms.
Examples
"Deposit money in the bank" vs "Sit by the river bank" generate distinct vectors for "bank".
Advantages
Captures deep, context-aware semantic nuances; sets the current state-of-the-art benchmark for down-stream tasks.
Limitations
Extremely high computational overhead; requires modern GPU/TPU infrastructure to run inference.
Applications
Modern LLMs, advanced conversational AI, precise semantic search.

2. Language Models

Definition
Probability distribution models over sequences of words constructed using classic Markov chains and raw frequency counting parameters.
Purpose
To calculate the absolute likelihood of a sequence of words occurring or to predict the next word in a chain.
Working Principle
Utilizes the chain rule of probability combined with N-gram counts to compute sequence likelihood, applying smoothing techniques to handle unseen text.
Examples
Calculating P("shines" | "the sun") based on text corpus counts.
Advantages
Incredibly fast inference; simple mathematical properties; easily interpretable.
Limitations
Suffers from the "curse of dimensionality"; cannot capture long-range contextual dependencies due to fixed history windows.
Applications
Early speech recognition systems, basic predictive keyboard typing systems.
Definition
Language modeling architectures that leverage continuous neural network layers (MLPs, RNNs, LSTMs) to learn word probability distributions.
Purpose
To overcome the sparsity and structural limits of SLMs by processing language within continuous vector spaces.
Working Principle
Passes sequence tokens through embedding layers, processing hidden structural states sequentially through recurrent network connections to predict tokens via Softmax.
Examples
An LSTM network processing a sequence word-by-word to generate predictive text patterns.
Advantages
Better generalization across hidden vectors; handles longer history contexts than statistical N-grams.
Limitations
Sequential processing cannot be parallelized during training stages; prone to vanishing gradients over long sequences.
Applications
Early-stage sequence generation systems, intermediate machine translation engines.
Definition
Advanced multi-headed self-attention network architectures that compute highly non-linear, parallelized language representations.
Purpose
To process long-range dependencies efficiently across an entire text sequence simultaneously.
Working Principle
Completely eliminates recurrent loops. Uses positional encodings to track order, and employs multi-head self-attention mechanisms to allow tokens to dynamically weight and connect with every other token in the sequence.
Examples
BERT (encoder-only for parsing) and GPT architecture variants (decoder-only for generative processing).
Advantages
Highly parallelizable across GPUs; captures long-range context across extensive text sequences.
Limitations
Computational and memory complexity scales quadratically ($O(L^2)$) relative to sequence length L without optimization.
Applications
State-of-the-art Large Language Models, semantic text synthesis, deep conversational AI interfaces.
