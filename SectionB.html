Architectural Comparative Analysis

Section B: Structural and Computational Flows with Downward Dependencies

Aarya Yugant Mhatre

1. Stemming vs Lemmatization

Stemming Node

Mechanism: Algorithmic truncation slicing word endings heuristics.
Complexity: Low; $O(n)$ operations.
Strengths: Fast; minimal storage footprint.
Weaknesses: Truncates to invalid terms (e.g., "studi").
Suitability: Search indices, high-speed lookup feeds.

Lemmatization Node

Mechanism: Full lexicon check using POS structural filters.
Complexity: High; dictionary query processing.
Strengths: Keeps valid dictionary base forms.
Weaknesses: Resource heavy; slower overall throughput.
Suitability: Chatbots, advanced question answering systems.
▼
Flow Transition Insight: Moving from Stemming to Lemmatization introduces structural context knowledge, transforming simple suffix removal operations into fully grammatical mapping nodes.

2. TF vs TF-IDF Flows

TF (Term Frequency) Node

Mechanism: Tracks localized document frequency counts.
Complexity: Linear $O(n)$ relative to text block length.
Strengths: Reflects internal theme weight directly.
Weaknesses: Biased heavily to common background tokens.
Suitability: Local density tracking maps.

TF-IDF Node

Mechanism: Modulates local frequency with log corpus rarity.
Complexity: Higher; requires global collection counts.
Strengths: Downweights global stop-words dynamically.
Weaknesses: High dimension vectors remain sparse.
Suitability: Search indexing retrieval loops.
▼
Flow Transition Insight: Adding inverse document statistics transforms local counts into global information metrics, isolating distinctive keyword patterns across extensive datasets.
Shraddha Sandip Salvi

3. SLM vs NLM

SLM Node

Mechanism: Markov chain parsing using raw N-gram frequencies.
Complexity: Low training math; fast hash lookups.
Strengths: High visibility math; fast generation tasks.
Weaknesses: Severe vocabulary sparsity constraints.
Suitability: Keyboard autocomplete loops.

NLM Node

Mechanism: Continuous deep matrix propagation pathways.
Complexity: High; intensive forward/backward propagation computations.
Strengths: Excellent generalization capacity across spaces.
Weaknesses: Low structural parameters transparency.
Suitability: Modern translation and conversational tasks.
▼
Flow Transition Insight: Shifting from statistical sequences to neural operations maps discrete tokens into continuous vector spaces, allowing models to generalize effectively across synonyms.

4. Word2Vec vs FastText Flows

Word2Vec Node

Mechanism: Predicts continuous coordinates for whole words.
Complexity: Moderate; bounded by core vocabulary indices.
Strengths: Great linear semantic analogy mappings.
Weaknesses: Fails entirely on out-of-vocabulary terms.
Suitability: Core word similarity frameworks.

FastText Node

Mechanism: Compiles and sums internal character subwords.
Complexity: High; tracks vast subword token tables.
Strengths: Resolves out-of-vocabulary terms and typos.
Weaknesses: High RAM footprints and storage overhead.
Suitability: Complex languages, error-prone user queries.
▼
Flow Transition Insight: Incorporating subword character context directly into the embedding layers allows models to construct meaningful vectors for previously unseen terms based on morphological clues.
